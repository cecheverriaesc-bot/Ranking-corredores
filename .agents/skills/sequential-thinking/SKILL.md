# Sequential Thinking — Instrucciones

Cuando el problema es complejo, ambiguo o crítico, ACTIVA este modo antes de responder.

## Protocolo de pensamiento secuencial

### Paso 1: Entender el problema
- ¿Cuál es exactamente el problema? (No lo que parece, sino lo que ES)
- ¿Qué información tengo? ¿Qué me falta?
- ¿Cuál es el impacto si no se resuelve?

### Paso 2: Hipótesis ordenadas
- Lista las posibles causas o soluciones en orden de probabilidad
- Empieza por las más simples (principio de Occam)
- Descarta opciones imposibles con evidencia

### Paso 3: Plan de verificación
- ¿Cómo verifico cuál hipótesis es correcta?
- ¿Qué prueba es más rápida de ejecutar?
- ¿Cuál tiene menor costo si está equivocada?

### Paso 4: Ejecutar y ajustar
- Ejecuta la prueba más informativa primero
- Documenta el resultado
- Si falla, avanza a la siguiente hipótesis

### Paso 5: Solución y aprendizaje
- Aplica la solución al problema original
- Documenta la causa raíz para futuras referencias

## Cuándo activar este modo
- Bugs que no tienen causa obvia
- Análisis de datos con resultados inesperados
- Decisiones de arquitectura con múltiples opciones
- Cualquier problema donde la primera solución intuitiva puede estar equivocada

## Regla de oro
**No respondas rápido. Responde bien.**
Si el problema vale la pena resolver, vale la pena pensar en él correctamente.
